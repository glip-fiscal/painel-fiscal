📚 Documentação do Projeto
GLIP · Extrator Fiscal Automatizado · v3.0
F1Salvar PDF no SharePoint✅ Ativo
Monitora [email protected] em tempo real. Ao receber e-mail, verifica os anexos e salva PDFs em /E-mails/Pendentes.
Trigger: Novo e-mail recebido (Outlook V3)
Ação: Loop nos anexos → verifica se é PDF → salva com nome original
F2Extrair Dados com Claude AI✅ Ativo
Detecta novo PDF em /Pendentes, envia para Claude API e extrai 15+ campos fiscais. Cria item na lista Lancamentos com Status = Pendente.
Trigger: Arquivo criado em /Pendentes
Modelo: claude-sonnet-4-20250514
Campos: tipo, fornecedor, CNPJ, NF, datas, valor, impostos, chave NFe
Proteção: Verifica duplicata por CodigoIntegracao antes de criar
F3Integração Omie✅ Ativo
Ao criar item na lista, busca credenciais no glip.json, faz upsert do fornecedor e lança a conta a pagar no Omie.
Trigger: Item criado na lista Lancamentos
Fluxo: UpsertClienteCpfCnpj → 60s atraso → IncluirContaPagar
Sucesso: 200, 425 ou 502 → move para /Processados
Erro: move para /Erro
Pendências manuais: Categoria (2.06.98) e Conta Corrente
F4Reconciliação e Painel✅ Ativo
Roda a cada hora. Lê notas da lista Lancamentos, detecta PDFs travados em /Pendentes e gera o arquivo diário no GitHub para o painel.
Trigger: Recorrência — a cada 1 hora
Filtro: DataRegistroHora1 da última hora
Saída: YYYY-MM-DD.json no GitHub com notas + pendentes
Deduplicação: por NF + CNPJ + DataEmissao

Resolvidos anteriormente
🔧 Claude retornava JSON com marcadores ```json
Solução: Adicionar ação Redigir antes do Analisar JSON com:
replace(replace(body('HTTP')?['content'][0]['text'], '```json', ''), '```', '')
🔧 Loop com paralelismo causava conflito de variáveis
Solução: Aplicar a cada → Configurações → Controle de Simultaneidade = 1
🔧 502 BadGateway no Omie
Solução: Tratar 502 como sucesso junto com 200 e 425 — o Omie processa mesmo retornando 502
🔧 Notas duplicadas na lista Lancamentos
Solução: No Fluxo 2, verificar CodigoIntegracao antes de criar item. No painel, deduplica por NF+CNPJ+DataEmissao
🔧 Horário das notas em UTC no painel
Solução: No Fluxo 4, usar addHours(DataRegistroHora1, -3) para converter para Brasília antes de salvar no JSON
🔧 Status vindo como objeto SharePoint
Solução: Usar items('Aplicar_a_cada')?['Status']?['Value'] em vez de ['Status']
🔧 trim() não suportado com 2 parâmetros no Power Automate
Solução: Substituir por substring(var, 0, sub(length(var), 1)) para remover última vírgula do array
Resolvidos em 21–23/03/2026
🔧 Fluxo 3 — multi-empresa: loop parava na 1ª iteração
Causa: Condição A (varEncontrado = false) estava dentro do "Aplicar a cada 1", bloqueando a 2ª iteração.
Solução: Mover a Condição A para depois do loop interno. Adicionar "Definir varEncontrado = false" antes do loop para reset a cada arquivo.
🔧 Fluxo 3 — arquivo não encontrado ao reprocessar nota com erro
Causa: "Mover arquivo" sempre buscava de /Pendentes, mas nota reprocessada estava em /Erro.
Solução: Trocar caminho fixo por concat('/E-mails/', if(greater(length(body('Obter_itens')?['value']),0),'Erro','Pendentes'), '/', NomeArquivo)
🔧 Fluxo 3 — item com Erro permanecia na lista após reprocessamento
Causa: Ao reprocessar com sucesso, o item antigo com Status=Erro não era removido da lista Lancamentos.
Solução: Após mover o PDF de /Erro para /Processados, adicionar "Excluir item" usando o ID do item com Erro retornado pelo "Obter itens".
🔧 Fluxo 3 — nota lançada duas vezes no Omie (duplicata)
Causa: HTTP do Omie com timeout ativava retentativa automática do Power Automate.
Solução: No bloco HTTP do Omie → Configurações → Política de repetição = Nenhuma.
🔧 Painel — arquivo JSON criado com data errada após meia-noite
Causa: varDataHoraAtual usava utcNow() — após 21h de Brasília já virava o dia seguinte em UTC.
Solução: Trocar por convertTimeZone(utcNow(), 'UTC', 'E. South America Standard Time', 'yyyy-MM-ddTHH:mm:ss')
🔧 Painel — erros de dias anteriores somiam ao filtrar por data
Causa: Quadro "Notas com Erro" lia do array notas do JSON diário, filtrado por data.
Solução: Criar erros.json separado, mantido pelo Fluxo 4 com todos os erros em aberto. Painel lê desse arquivo independente da data selecionada.
🔧 Painel — filtro de empresa zerava ao trocar de data sem lançamentos
Causa: Dropdown de empresas era populado pelas notas do dia — dias sem lançamento ficavam sem opções.
Solução: Criar glip.json no GitHub com lista de empresas cadastradas. Painel carrega empresas desse arquivo ao iniciar.
🔧 Painel — PDFs pendentes sumiam ao trocar de data
Causa: Quadro "PDFs Pendentes" lia do JSON do dia selecionado.
Solução: Criar pendentes.json separado, mantido pelo Fluxo 4 com estado atual da pasta /Pendentes. Painel lê desse arquivo independente da data.
🔧 Fluxo 3 — razao_social com mais de 60 caracteres causa erro 502 vazio (23/03/2026)
Causa: Omie limita razao_social a 60 caracteres. Quando excede, retorna SOAP-ENV:Client-7002 via 502 com body vazio, deixando varCodigoFornecedor vazio e quebrando o HTTP 2 com InvalidTemplate.
Solução: No body do HTTP 1, usar substring(triggerBody()?['NomeFornecedor'], 0, min(length(triggerBody()?['NomeFornecedor']), 60))
🔧 Fluxo 3 — glip.json no SharePoint não sincronizado com painel (23/03/2026)
Causa: Empresas cadastradas no painel salvavam no GitHub mas o Fluxo 3 lia do SharePoint — dois arquivos diferentes.
Solução: Substituir "Enviar uma solicitação HTTP ao SharePoint" por HTTP GET simples para https://raw.githubusercontent.com/glip-fiscal/painel-fiscal/main/glip.json. Atualizar Analisar JSON 1 para usar body('HTTP').
Clique em um item para marcar como concluído. Items concluídos ficam riscados.
